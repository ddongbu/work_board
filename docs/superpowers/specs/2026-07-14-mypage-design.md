## 개요

개인 기술 블로그의 사용자 설정 페이지. 벨로그 스타일 레이아웃을 참고하여 프로필 관리와 계정 설정을 단일 페이지에서 처리한다.

---

## 기능 범위

| 항목 | 내용 |
|---|---|
| 프로필 사진 | AWS S3 업로드(`profiles/` 경로), 이미지 업로드 / 제거 |
| 닉네임 변경 | 인라인 수정, 중복 확인 |
| 이메일 표시 | 읽기 전용 |
| 비밀번호 변경 | 현재 비밀번호 확인 후 새 비밀번호 교체 |
| 회원 탈퇴 | 확인 모달 후 계정 삭제 |

---

## 헤더 드롭다운

로그인 상태에서 우측 프로필 이미지(원형) + 화살표(`▼`) 클릭 시 드롭다운 표시.

```
[ 프로필 사진 ▼ ]
─────────────────
  설정
  로그아웃
```

- 프로필 사진 미설정 시 기존 닉네임 첫 글자 색깔 아바타 유지
- 설정 클릭 → `/settings` 이동
- 로그아웃 클릭 → 기존 logout 로직 실행

---

## 설정 페이지 레이아웃 (`/settings`)

벨로그 스타일 단일 페이지. 섹션을 수평선(`<hr>`)으로 구분.

```
┌─────────────────────────────────────────────────┐
│  [프로필 사진 원형]                               │
│  [이미지 업로드 버튼]  [이미지 제거 링크]           │
├─────────────────────────────────────────────────┤
│  닉네임          Danny                   [수정]  │
├─────────────────────────────────────────────────┤
│  이메일          sang214q@vdream.co.kr  (표시만) │
├─────────────────────────────────────────────────┤
│  비밀번호 변경                                    │
│  현재 비밀번호   [                    ]           │
│  새 비밀번호     [                    ]           │
│  비밀번호 확인   [                    ]           │
│                                         [변경]   │
├─────────────────────────────────────────────────┤
│  회원 탈퇴       [회원 탈퇴] ← red 버튼            │
│  탈퇴 시 작성한 게시글과 댓글이 모두 삭제됩니다.    │
└─────────────────────────────────────────────────┘
```

- 비로그인 접근 시 `/`로 리다이렉트
- 닉네임 수정: [수정] 클릭 → 인풋 활성화 → [저장] / [취소]
- 프로필 사진 변경 성공 시 `authStore` 즉시 업데이트 → 헤더 반영
- 회원 탈퇴: 확인 모달("정말 탈퇴하시겠습니까?") 후 실행

## 백엔드 변경

### 모델 (`app/core/models.py`)

`User` 모델에 컬럼 추가:
```python
profile_image_url: Mapped[str | None] = mapped_column(VARCHAR(1000), nullable=True)
```

### 신규 도메인 `app/api/mypage/`

| 메서드 | 엔드포인트 | 설명 |
|---|---|---|
| `PUT` | `/mypage/profile` | 닉네임 + 프로필 사진 URL 변경 |
| `PUT` | `/mypage/password` | 비밀번호 변경 (현재 비밀번호 확인 필수) |
| `DELETE` | `/mypage/account` | 회원 탈퇴 |

### 기존 엔드포인트 수정

- `GET /auth/me` 응답에 `profile_image_url` 포함
- `POST /upload` — S3 키 경로를 요청 파라미터(`folder`)로 분기 가능하도록 수정 (`posts/` vs `profiles/`)

## 데이터 흐름

```
사용자 → 이미지 선택
  → POST /upload?folder=profiles (기존 엔드포인트)
  → S3 profiles/{uuid}.{ext} 저장
  → URL 반환
  → PUT /mypage/profile { profile_image_url: url }
  → authStore 업데이트
  → 헤더 즉시 반영
```
